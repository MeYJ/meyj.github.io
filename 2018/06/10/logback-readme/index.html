logback 使用配置小记

Posted on 2018-06-10 |

配置介绍

logback 配置主要分为三大块：

  • logger
  • appender
  • layout

logger

日志记录器，主要用于关联日志对象，同时还可以定义日志的「类型」、「级别」

appender

日志输出目的地，常用的有「控制台」，「文件」，「syslog」

layout

日志输出格式，负责将日志按照制定的格式输出

日志级别排序

日志级别从低到高依次为：TRACE < DEBUG < INFO < WARN < ERROR

定义的级别越低，输出的日志内容越多。使用最多的是 DEBUG（调试信息），INFO（普通提示），ERROR（错误消息） 三个级别。

配置文件

logback 的配置文件为 logback.xml，只要将此配置文件放置在程序的 classpath 目录下，使用时 logback 会自动加载此配置。

logback 使用

日志框架的使用通常都会借助「日志门面」来进行代码上的解耦，以防止程序与某一个具体的日志框架耦合过紧。目前最流行的「日志门面」当然非 slf4j 莫属，同时它也是与 logback 配合最好的「日志门面」，不需要任何桥接程序就能使用。

使用时只要将 slf4j 与 logback 相关的 jar 包放置到 classpath 目录下，代码中利用 slf4j 的 Logger 接口定义日志输出对象并用其输出日志，实现「面向接口」输出日志。

slf4j 会自动加载并关联具体的日志框架，程序不需要关心具体日志框架，实现日志框架的完全解耦，方便随时替换。
